# 人情世故助手 (Social Wisdom Assistant) Spec

## Why
很多人在社交场合难以理解他人言语背后的真实意图，无法听出隐喻和弦外之音，导致应对失当、关系受损。这是一个AI驱动的桌面应用，帮助用户实时分析对话中的潜台词，并提供恰当的回复建议。

## What Changes
- 构建跨平台桌面应用（支持Windows/macOS）
- **Docker容器化部署，一键启动，环境隔离**
- 实现多种输入方式：截屏分析、实时语音捕获、录音文件上传
- 接入多模态大模型，提供对话意图分析和回复建议
- 构建人情世故知识库，增强AI的社交洞察力
- **自我进化学习系统：自动从抖音等平台抓取社交技巧视频，提取知识入库**

## Impact
- Affected specs: 全新项目，无现有spec
- Affected code: 全新代码库

## ADDED Requirements

### Requirement: Docker容器化部署
系统应支持Docker一键部署，简化环境配置。

#### Scenario: 一键启动
- **WHEN** 用户执行 `docker-compose up -d`
- **THEN** 系统自动拉取镜像、配置环境、启动所有服务
- **AND** 用户无需手动安装Node.js、Rust、Python等依赖

#### Scenario: 环境隔离
- **WHEN** 应用运行在Docker容器中
- **THEN** 所有依赖和运行时环境完全隔离
- **AND** 不会与用户系统环境冲突

#### Scenario: 跨平台一致性
- **WHEN** 用户在不同操作系统上运行
- **THEN** Docker确保运行环境完全一致
- **AND** 消除"在我机器上能跑"的问题

### Requirement: 自我进化学习系统
系统应能自动从互联网学习社交知识，持续进化。

#### Scenario: 视频内容抓取
- **WHEN** 系统执行定时学习任务或用户手动触发
- **THEN** 系统从抖音/B站等平台抓取社交技巧相关视频
- **AND** 支持关键词配置（如"职场话术"、"领导意图"、"官场智慧"等）

#### Scenario: 视频内容提取
- **WHEN** 视频下载完成
- **THEN** 系统自动提取视频中的语音转文字
- **AND** 提取视频字幕（如有）
- **AND** 使用多模态模型分析视频画面内容

#### Scenario: 知识入库
- **WHEN** 视频内容提取完成
- **THEN** AI分析提取社交技巧知识点
- **AND** 结构化存储到知识库
- **AND** 建立知识索引便于检索

#### Scenario: 学习进度追踪
- **WHEN** 用户查看学习记录
- **THEN** 显示已学习的视频数量、提取的知识点数量
- **AND** 显示最近学习的内容摘要

### Requirement: 多模态输入支持
系统应支持多种输入方式以适应不同场景。

#### Scenario: 截屏分析
- **WHEN** 用户按下快捷键截屏
- **THEN** 系统捕获屏幕内容并发送给多模态模型分析

#### Scenario: 实时语音捕获
- **WHEN** 用户开启实时监听模式
- **THEN** 系统持续捕获麦克风音频和系统音频（如会议软件）
- **AND** 实时转写并分析对话内容

#### Scenario: 录音文件上传
- **WHEN** 用户上传录音文件（支持手机同步）
- **THEN** 系统转写音频并分析对话内容

### Requirement: 对话意图分析
系统应能分析对话中的深层含义。

#### Scenario: 识别隐喻和潜台词
- **WHEN** 用户提交一段对话
- **THEN** 系统分析并输出：
  - 说话人的真实意图
  - 话语中的隐喻和暗示
  - 情绪状态和态度
  - 可能的社交陷阱

#### Scenario: 提供回复建议
- **WHEN** 分析完成
- **THEN** 系统提供多个回复选项：
  - 得体回应方案
  - 拒绝/推脱话术
  - 进一步追问建议
  - 每个选项的利弊分析

### Requirement: 人情世故知识库
系统应具备丰富的社交知识支持。

#### Scenario: 知识库检索
- **WHEN** 分析对话时
- **THEN** 系统从知识库中检索相关社交规则、心理学原理、文化背景
- **AND** 结合知识库内容给出更准确的分析

### Requirement: 手机端实时助手（第二阶段）
系统应提供独立的手机端应用，实现实时语音分析。

#### Scenario: 实时监听与分析
- **WHEN** 用户在手机上开启实时监听模式
- **THEN** 手机持续捕获语音并实时转写为文字
- **AND** 转写内容实时发送给大模型分析
- **AND** 分析结果和回复建议实时返回显示在手机屏幕上

### Requirement: 隐私保护
系统应保护用户隐私。

#### Scenario: 本地处理优先
- **WHEN** 用户选择本地模式
- **THEN** 敏感数据优先在本地处理
- **AND** 用户可控制数据是否上传到云端

### 技术栈详情

#### 前端服务 (Frontend)
- **框架**: React + TypeScript + TailwindCSS
- **状态管理**: Zustand
- **UI组件**: Shadcn/ui
- **特点**: 轻量、响应式、现代化界面

#### 后端服务 (Backend)
- **框架**: Python + FastAPI
- **功能**: 
  - API网关
  - 对话分析逻辑
  - 知识库检索 (RAG)
  - 多模型统一接口

#### 学习服务 (Learning Service)
- **框架**: Python + Celery + Redis
- **视频抓取**: 参考 [TikTokDownloader](https://github.com/JoeanAmier/TikTokDownloader)
- **视频分析**: 参考 [Video-Analyzer](https://github.com/LLM-Red-Team/Video-Analyzer)
- **语音转写**: OpenAI Whisper (本地) 或 云端ASR
- **功能**:
  - 定时抓取指定关键词的视频
  - 提取视频语音和字幕
  - AI分析提取知识点
  - 自动入库

#### AI服务
- **本地模型**: Ollama + Qwen2-VL / Llama3.2-Vision
- **云端模型**: OpenAI API / 通义千问 / 文心一言
- **语音转写**: Whisper (本地) 或 云端ASR

#### 数据层
- **PostgreSQL**: 用户数据、学习记录、配置
- **ChromaDB**: 向量知识库，支持语义检索
- **Redis**: 任务队列、缓存
- **MinIO**: 视频、音频文件存储

## MVP范围

### 第一阶段（MVP - 桌面端核心功能）
1. Docker一键部署
2. 截屏分析（接入多模态模型）
3. 文本输入分析
4. 基础知识库（手动导入）
5. 回复建议生成
6. 基础UI界面

### 第二阶段（自我进化学习）
1. 视频抓取模块（抖音/B站）
2. 视频内容提取（语音+字幕）
3. AI知识点提取
4. 自动入库
5. 学习进度追踪

### 第三阶段（音频功能）
1. 实时语音捕获和转写
2. 录音文件上传分析
3. 手机端实时助手

## Out of Scope
- 视频实时分析（后续版本考虑）
- 社交媒体直接集成
- 用户账号系统和云同步（MVP阶段）
- 付费功能和商业化
